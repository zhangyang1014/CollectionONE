# ⚠️ 自动启动失败 - 网络代理问题

## 🔍 问题确认

由于系统级别的网络代理配置，无法通过命令行自动安装Python依赖包。

## ✅ 必须手动操作

### 步骤1：打开新终端窗口

### 步骤2：执行以下命令（复制粘贴）

```bash
cd "/Users/zhangyang/Library/Mobile Documents/com~apple~CloudDocs/2. 领域（Areas）/17 学习AI 与 编程/Code/CloudunCollectionONE/backend"

# 激活虚拟环境
source venv/bin/activate

# 方法A：禁用代理后安装（推荐）
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
pip install -r requirements.txt

# 方法B：如果方法A不行，使用国内镜像
# pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 启动服务
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤3：等待看到启动成功

```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 步骤4：验证API

打开浏览器：`http://localhost:8000/docs`

搜索 `performance`，应该能看到数据看板API端点。

### 步骤5：刷新前端

按 `Cmd + Shift + R` 强制刷新，然后点击"数据看板" → "我的业绩看板"。

---

## 📝 如果安装依赖时还是报代理错误

请检查系统代理设置：
1. **macOS系统设置** → **网络** → **高级** → **代理**
2. 临时禁用HTTP/HTTPS代理
3. 或者配置pip使用系统代理的正确地址
