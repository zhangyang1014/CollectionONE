# CCO产品功能清单

**版本：** v1.0  
**更新时间：** 2025-11-12  
**系统名称：** CCO（催收运营系统）

---

## 一、系统架构

### 1.1 系统组成

```
CCO系统
├── 管理控台端（Admin Portal）- 管理、配置、监控
└── IM端（Instant Messaging）- 催员工作台
```

### 1.2 技术栈

- **前端：** Vue 3 + TypeScript + Element Plus
- **后端：** Python + FastAPI + SQLAlchemy
- **数据库：** SQLite/MySQL
- **特性：** 多租户、多语言、多时区

---

## 二、角色清单

### 2.1 角色层级架构

```
超级管理员（SuperAdmin）
  └── 甲方管理员（TenantAdmin）
      └── 机构管理员（AgencyAdmin）
          └── 小组长（TeamLeader）
              ├── 质检员（QualityInspector）
              ├── 数据源（DataSource/Statistician）
              └── 催员（Collector）
```

### 2.2 角色详细说明

| 角色 | 英文名称 | 主要职责 | 登录端 |
|------|---------|---------|--------|
| 超级管理员 | SuperAdmin | 系统最高权限，管理甲方、标准字段、系统配置 | 控台端 |
| 甲方管理员 | TenantAdmin | 管理本甲方的所有配置、组织架构、案件队列 | 控台端 |
| 机构管理员 | AgencyAdmin | 管理机构、小组、催员、查看机构案件和业绩 | 控台端 |
| 小组长 | TeamLeader | 管理小组催收工作、查看组员聊天和业绩 | 控台端 |
| 质检员 | QualityInspector | 质检催员聊天记录、评分、生成质检报告 | 控台端 |
| 数据源/统计员 | DataSource | 查看业绩数据、生成报表、数据分析 | 控台端 |
| 催员 | Collector | 执行催收工作、联系客户、记录催收活动 | IM端 |
