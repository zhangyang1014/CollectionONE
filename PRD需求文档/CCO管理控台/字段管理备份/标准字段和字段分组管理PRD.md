# 标准字段和字段分组管理 PRD

## 一、产品需求（Product Requirements）

### 1. 项目背景与目标（Background & Goals）

标准字段和字段分组管理是CCO管理控台的基础功能之一，用于定义系统核心字段集合和字段分类体系。标准字段是系统官方定义的基础字段集合，所有甲方自动继承，是字段映射和展示配置的基础。字段分组用于对字段进行分类管理，支持多级分组结构，便于字段的组织和查找。通过统一的标准字段管理，确保系统字段定义的一致性和规范性，为后续的字段映射、展示配置等功能提供基础支撑。

**业务痛点**：
- 不同甲方字段命名和结构差异大，需要统一的字段标准
- 字段数量庞大，需要分类管理便于查找和维护
- 字段配置变更频繁，需要版本管理和变更追踪
- 标准字段定义不清晰，导致字段映射困难
- 字段分组结构混乱，影响字段查找效率

**预期影响的核心指标**：
- 字段配置效率：标准字段配置完成时间≤5分钟/字段
- 字段查找效率：通过分组查找字段时间≤10秒
- 字段一致性：标准字段定义准确率≥99%
- 用户满意度：管理员对标准字段管理功能满意度≥90%

#### 2.1 典型使用场景

**场景1：查看标准字段列表**

- **入口**：标准字段管理页面 → 点击"添加字段"按钮

- **所在页面**：标准字段管理页面（弹窗）
- **流程节点**：填写字段信息 → 选择字段分组 → 配置字段属性 → 保存

**场景3：编辑标准字段**

- **触发时机**：需要修改标准字段属性时
- **所在页面**：标准字段管理页面（弹窗）

- **触发时机**：需要调整字段分组结构或排序时
- **所在页面**：`/field-config/groups`
- **流程节点**：查看分组树 → 添加/编辑/删除分组 → 拖拽调整排序

**场景5：拖拽字段排序**
- **入口**：标准字段管理页面 → 拖拽字段行
- **触发时机**：需要调整字段显示顺序时
- **所在页面**：标准字段管理页面

### 3. 关键业务流程（Business Flow）

#### 3.1 标准字段管理流程

```
管理员进入标准字段管理页面
    ↓
加载字段分组树（左侧）
    ↓
自动选中第一个分组
    ↓
加载该分组下的标准字段列表（右侧）
    ↓
支持操作：
  - 添加字段：点击"添加字段" → 填写字段信息 → 保存
  - 编辑字段：点击"编辑" → 修改字段信息 → 保存
  - 删除字段：点击"删除" → 确认删除（软删除）
  - 拖拽排序：拖拽字段行 → 自动保存排序
  - 启用/禁用：切换开关 → 自动保存
```

#### 3.2 添加标准字段流程

```
管理员点击"添加字段"按钮
    ↓
打开添加字段弹窗
    ↓
填写字段信息：
  - 字段标识（field_key）：必填，唯一，小写字母+下划线
  - 字段名称（field_name）：必填，中文名称
  - 字段名称（英文）：可选
  - 字段类型（field_type）：必填，下拉选择
  - 所属分组（field_group_id）：必填，下拉选择
  - 是否必填（is_required）：开关，默认false
  - 字段说明（description）：可选
  - 示例值（example_value）：可选
  - 枚举选项（enum_options）：仅Enum类型需要
    ↓
点击"保存"按钮
    ↓
校验字段信息：
  - 字段标识唯一性检查
  - 字段类型校验
  - 枚举值格式校验（如果是Enum类型）
    ↓
[校验失败] → 显示错误提示
    ↓
[校验成功] → 保存到数据库
    ↓
刷新字段列表
```

#### 3.3 编辑标准字段流程

```
管理员点击"编辑"按钮
    ↓
打开编辑字段弹窗（预填充字段信息）
    ↓
修改字段信息（字段标识不可修改）
    ↓
点击"保存"按钮
    ↓
校验字段信息
    ↓
[校验失败] → 显示错误提示
    ↓
[校验成功] → 更新数据库
    ↓
刷新字段列表
```

#### 3.4 字段分组管理流程

```
管理员进入字段分组管理页面
    ↓
加载字段分组树（支持父子分组）
    ↓
支持操作：
  - 添加一级分组：点击"添加一级分组" → 填写分组信息 → 保存
  - 添加子分组：选中分组 → 点击"添加子分组" → 填写分组信息 → 保存
  - 编辑分组：点击"编辑" → 修改分组信息 → 保存
  - 删除分组：点击"删除" → 检查是否有字段关联 → 确认删除
  - 拖拽排序：拖拽分组节点 → 自动保存排序
```

#### 3.5 添加字段分组流程

```
管理员点击"添加分组"按钮
    ↓
打开添加分组弹窗
    ↓
填写分组信息：
  - 分组标识（group_key）：必填，唯一，小写字母+下划线
  - 分组名称（group_name）：必填，中文名称
  - 分组名称（英文）：可选
  - 父分组（parent_id）：可选，下拉选择
  - 排序顺序（sort_order）：数字，默认0
    ↓
点击"保存"按钮
    ↓
校验分组信息：
  - 分组标识唯一性检查
  - 父分组存在性检查
    ↓
[校验失败] → 显示错误提示
    ↓
[校验成功] → 保存到数据库
    ↓
刷新分组树
```

#### 3.6 删除字段分组流程

```
管理员点击"删除"按钮
    ↓
检查是否有字段关联该分组
    ↓
[有字段关联] → 提示"该分组下有关联字段，无法删除"

```

### 4. 业务规则与边界（Business Rules & Scope）

#### 4.1 标准字段管理规则

**字段属性规则**：
- **字段标识（field_key）**：
  - 必填，全局唯一
  - 只能包含小写字母、数字和下划线
  - 必须以小写字母开头
  - 长度限制：1-100个字符
  - 创建后不可修改
- **字段名称（field_name）**：
  - 必填，中文名称
  - 长度限制：1-200个字符
  - 可编辑
- **字段类型（field_type）**：
  - 必填，必须是以下类型之一：
    - `String`：文本类型
    - `Integer`：整数类型
    - `Decimal`：小数类型
    - `Date`：日期类型（格式：YYYY-MM-DD）
    - `Datetime`：日期时间类型（格式：ISO8601）
    - `Boolean`：布尔类型（true/false）
    - `Enum`：枚举类型（需提供enum_options）
  - 可编辑（需检查是否有数据依赖）
- **所属分组（field_group_id）**：
  - 必填，必须在系统中存在
  - 可编辑
- **是否必填（is_required）**：
  - 默认false
  - 可编辑
- **是否拓展字段（is_extended）**：
  - 标准字段固定为false
  - 不可编辑
- **排序顺序（sort_order）**：
  - 数字，支持拖拽自动更新
  - 可编辑

**操作规则**：
- 标准字段可以编辑（字段名称、类型、分组、必填等）
- 标准字段可以启用/禁用（is_active）
- 标准字段可以删除（软删除，is_deleted=true）
- 标准字段支持拖拽排序，自动更新sort_order
- 字段标识创建后不可修改

**范围边界**：
- ✅ 支持字段的增删改查
- ✅ 支持字段分组管理
- ✅ 支持字段排序
- ✅ 支持字段启用/禁用
- ❌ 不支持字段值的编辑（字段值在案件数据中管理）
- ❌ 不支持字段标识的修改（创建后不可修改）

#### 4.2 字段分组管理规则

**分组属性规则**：
- **分组标识（group_key）**：
  - 必填，全局唯一
  - 只能包含小写字母、数字和下划线
  - 长度限制：1-100个字符
  - 创建后不可修改
- **分组名称（group_name）**：
  - 必填，中文名称
  - 长度限制：1-200个字符
  - 可编辑
- **父分组（parent_id）**：
  - 可选，支持多级分组
  - 不能选择自己作为父分组
  - 不能形成循环引用
  - 可编辑
- **排序顺序（sort_order）**：
  - 数字，支持拖拽自动更新
  - 可编辑

**操作规则**：
- 支持添加一级分组和子分组
- 支持编辑分组信息（分组标识不可修改）
- 支持删除分组（需检查是否有字段关联）
- 支持拖拽调整分组顺序和层级
- 分组标识创建后不可修改

**删除规则**：
- 删除分组前必须检查是否有字段关联该分组
- 如果有关联字段，提示"该分组下有关联字段，无法删除"
- 如果无关联字段，可以删除（软删除）

**范围边界**：
- ✅ 支持分组的增删改查
- ✅ 支持多级分组（父子关系）
- ✅ 支持分组排序
- ✅ 支持分组启用/禁用
- ❌ 不支持分组合并
- ❌ 不支持分组标识的修改（创建后不可修改）

#### 4.3 字段类型规则

**支持的字段类型**：
- `String`：文本类型，无特殊要求
- `Integer`：整数类型，无特殊要求
- `Decimal`：小数类型，无特殊要求
- `Date`：日期类型，格式：YYYY-MM-DD
- `Datetime`：日期时间类型，格式：ISO8601
- `Boolean`：布尔类型，值：true/false
- `Enum`：枚举类型，必须提供enum_options数组

**枚举值格式规则**：
- `enum_options`必须是数组
- 每个枚举项必须包含：
  - `value`：枚举值（必填）
  - `label`：枚举标签（必填）
  - `standard_name`：标准名称（可选）
- 示例：
```json
{
  "enum_options": [
    {"value": "PENDING", "label": "待还款", "standard_name": "待还款"},
    {"value": "PARTIAL", "label": "部分还款", "standard_name": "部分还款"},
    {"value": "SETTLED", "label": "正常结清", "standard_name": "正常结清"}
  ]
}
```

- 记录操作人、操作时间、操作内容

### 1. 系统架构与模块划分（System Architecture & Modules）

#### 1.1 模块划分

```
标准字段和字段分组管理模块
│
├── 标准字段管理模块
```

### 2. 接口设计与系统依赖（API Design & Dependencies）

#### 2.1 标准字段管理接口

**2.1.1 获取标准字段列表**
- **接口**：`GET /api/v1/standard-fields`
- **请求参数**：
  - `field_group_id`（可选）：字段分组ID，用于筛选
  - `is_active`（可选）：是否启用，用于筛选
- **响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "field_key": "user_id",
      "field_name": "用户ID",
      "field_name_en": "User ID",
      "field_type": "String",
      "field_group_id": 1,
      "is_required": true,
      "is_extended": false,
      "description": "用户的唯一标识",
      "example_value": "12345678",
      "enum_options": null,
      "sort_order": 1,
      "is_active": true
    }
  ]
}
```

**2.1.2 创建标准字段**
- **接口**：`POST /api/v1/standard-fields`
- **请求体**：
```json
{
  "field_key": "new_field",
  "field_name": "新字段",
  "field_name_en": "New Field",
  "field_type": "String",
  "field_group_id": 1,
  "is_required": false,
  "description": "字段说明",
  "example_value": "示例值",
  "enum_options": null,
  "sort_order": 10
}
```
- **响应格式**：同获取接口

**2.1.3 更新标准字段**
- **接口**：`PUT /api/v1/standard-fields/{id}`
- **请求体**：同创建接口（field_key不可修改）
- **响应格式**：同获取接口

**2.1.4 删除标准字段**
- **接口**：`DELETE /api/v1/standard-fields/{id}`
- **响应格式**：
```json
{
  "code": 200,
  "message": "删除成功"
}
```

**2.1.5 批量更新字段排序**
- **接口**：`PUT /api/v1/standard-fields/sort`
- **请求体**：
```json
{
  "sorts": [
    {"id": 1, "sort_order": 1},
    {"id": 2, "sort_order": 2}
  ]
}
```

**2.1.6 启用/禁用字段**
- **接口**：`PUT /api/v1/standard-fields/{id}/toggle-active`
- **请求体**：
```json
{
  "is_active": true
}
```

#### 2.2 字段分组管理接口

**2.2.1 获取字段分组列表**
- **接口**：`GET /api/v1/field-groups`
- **响应格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "group_key": "customer_basic",
      "group_name": "客户基础信息",
      "group_name_en": "Customer Basic Info",
      "parent_id": null,
      "sort_order": 1,
      "is_active": true,
      "children": [
        {
          "id": 2,
          "group_key": "customer_contact",
          "group_name": "联系信息",
          "parent_id": 1,
          "sort_order": 1
        }
      ]
    }
  ]
}
```

**2.2.2 创建字段分组**
- **接口**：`POST /api/v1/field-groups`
- **请求体**：
```json
{
  "group_key": "new_group",
  "group_name": "新分组",
  "group_name_en": "New Group",
  "parent_id": null,
  "sort_order": 10
}
```

**2.2.3 更新字段分组**
- **接口**：`PUT /api/v1/field-groups/{id}`
- **请求体**：同创建接口（group_key不可修改）

**2.2.4 删除字段分组**
- **接口**：`DELETE /api/v1/field-groups/{id}`
- **前置检查**：检查是否有字段关联该分组
- **响应格式**：
```json
{
  "code": 200,
  "message": "删除成功"
}
```

**2.2.5 批量更新分组排序**
- **接口**：`PUT /api/v1/field-groups/sort`
- **请求体**：
```json
{
  "sorts": [
    {"id": 1, "sort_order": 1, "parent_id": null},
    {"id": 2, "sort_order": 2, "parent_id": 1}
  ]
}
```

#### 2.3 接口依赖

**内部依赖**：
- 无

**外部依赖**：
- 无

#### 2.4 超时与重试策略

- **超时时间**：5秒
- **重试策略**：不重试
- **幂等要求**：创建和更新接口需要幂等

#### 2.5 失败降级逻辑

- **创建失败**：返回错误信息，不保存
- **更新失败**：返回错误信息，不更新数据库
- **删除失败**：返回错误信息，不删除数据

### 4. 非功能性要求（Non-Functional Requirements）

#### 4.1 性能目标

- **接口响应时间**：所有接口响应时间≤200ms
- **字段列表加载**：加载100个字段≤500ms
- **分组树加载**：加载50个分组≤300ms

#### 4.2 可用性要求

- **SLA**：接口可用性≥99.9%
- **降级策略**：操作失败时返回详细错误信息

#### 4.3 安全要求

- **接口鉴权**：所有接口需要Token验证
- **权限控制**：仅SuperAdmin可以操作
