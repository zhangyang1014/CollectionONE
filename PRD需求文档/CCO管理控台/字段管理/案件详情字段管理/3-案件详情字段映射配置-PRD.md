## 1. 背景与目标

### 1.1 业务背景

字段映射配置是CCO管理控台的核心功能，用于将甲方通过API推送或上传的字段与系统标准字段建立映射关系。案件详情场景以**分组为核心**，支持在全部视图和分组视图下进行字段映射配置。

### 1.2 核心目标

1. **统一数据标准**：将甲方多样化的字段（含分组）映射到系统标准字段
2. **灵活扩展**：对无法映射到标准字段的甲方特有字段，支持创建拓展字段
3. **分组管理**：支持在分组视图下配置，提升配置效率
4. **版本控制**：支持甲方字段的多版本管理，可随时切换和对比
5. **自动匹配**：提供智能匹配算法，减少人工配置工作量

### 1.3 业务痛点

- 案件详情字段多且复杂，需要分组管理提升配置效率
- 不同甲方字段命名和分组结构差异大，需要灵活的映射机制
- 字段映射配置复杂，需要高效的匹配和配置工具
- 未映射字段可能导致数据丢失，需要强提醒机制
- 字段映射关系变更频繁，需要版本管理和变更追踪

### 1.4 预期指标

- 字段映射效率：单个字段映射完成时间 ≤ 30秒
- 自动匹配准确率：自动匹配建议准确率 ≥ 80%
- 字段映射覆盖率：必填字段映射覆盖率 = 100%
- 未映射字段处理率：未映射字段处理率 ≥ 95%
- 用户满意度：管理员对字段映射功能满意度 ≥ 90%

## 3. 数据源说明

### 3.1 数据源构成

案件详情字段映射配置涉及**两个核心数据源**（含分组结构）：

```text
字段映射数据源
│
├── 数据源1：标准字段（系统统一配置，含分组）
│   ├── 来源：系统内置，所有甲方继承
│   ├── 分组数：5个（基本信息、借款信息、催收信息、联系信息、产品信息）
│   ├── 字段数：约21个
│   ├── 接口：GET /api/v1/standard-fields/case-detail
│   ├── 特点：只读，不可编辑，不可删除，分组结构固定
│   └── 用途：作为映射的"目标字段"
│
└── 数据源2：甲方字段（甲方上传的字段配置，含分组）
    ├── 来源：甲方上传JSON文件
    ├── 分组数：可变（通常3-8个）
    ├── 字段数：可变（通常15-50个）
    ├── 接口：GET /api/v1/tenants/{tenantId}/fields-json?scene=detail
    ├── 特点：支持多版本管理，可随时切换版本，含分组结构
    └── 用途：作为映射的"源字段"
```

### 3.2 标准字段详情（含分组）

系统提供**5个标准分组，约21个标准字段**：

**分组1：基本信息** (4个字段)
- case_code - 案件编号
- user_name - 客户姓名
- id_card - 身份证号
- case_status - 案件状态

**分组2：借款信息** (7个字段)
- loan_amount - 贷款金额
- outstanding_amount - 未还金额
- overdue_days - 逾期天数
- due_date - 到期日期
- total_installments - 期数
- term_days - 当期天数
- loan_id - 关联借款ID

**分组3：催收信息** (3个字段)
- collector_name - 催收员姓名
- collection_stage - 催收阶段
- last_contact_time - 最后联系时间

**分组4：联系信息** (3个字段)
- mobile_number - 手机号
- emergency_contact - 紧急联系人
- emergency_phone - 紧急联系电话

**分组5：产品信息** (4个字段)
- system_name - 所属系统
- product_name - 产品
- app_name - APP
- merchant_name - 商户

### 3.3 甲方字段详情（含分组）

甲方通过上传JSON文件配置字段，必须包含分组结构，支持**多版本管理**。

**分组节点说明**：
- 分组名称（如"基本信息"）

- 完成率（如"100%"）
- 颜色标识：
  - 绿色：100%完成
  - 橙色：50-99%完成
  - 红色：<50%完成

**交互规则**：
- 点击"全部视图"：右侧显示所有标准字段（不分组），支持跨分组映射
- 点击"分组视图"：显示分组树，默认选中第一个分组
- 点击分组节点：右侧表格仅显示该分组下的标准字段
- 支持分组折叠/展开

## 7. 接口设计

### 7.1 获取标准字段（含分组）

**接口**：GET /api/v1/standard-fields/case-detail

**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "groups": [
      {
        "group_id": 1,
        "group_name": "基本信息",
        "group_key": "basic_info",
        "sort_order": 1,
        "fields": [
          {
            "field_key": "case_code",
            "field_name": "案件编号",
            "field_data_type": "String",
            "is_required": true
          }
        ]
      }
    ]
  }
}
```

### 7.2 智能匹配建议（支持分组范围）

**接口**：POST /api/v1/tenants/{tenantId}/field-configs/auto-suggest

**请求参数**：
```json
{
  "scene": "detail",
  "scope": "all",  // all：全部字段，group：指定分组，unmapped：未映射字段
  "group_key": "basic_info"  // 当scope=group时必填
}
```

**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "suggestions_by_group": {
      "basic_info": [
        {
          "field_key": "case_code",
          "tenant_field_key": "CASE_ID",
          "similarity": 100,
          "match_type": "exact"
        }
      ],
      "loan_info": [...]
    },
    "summary": {
      "total": 15,
      "high_confidence": 10,
      "medium_confidence": 5,
      "groups_count": 3
    }
  }
}
```

### 7.3 保存映射配置（含分组信息）

**接口**：POST /api/v1/field-mapping/save

**请求参数**：
```json
{
  "tenant_id": 1,
  "scene": "case_detail",
  "field_mappings": [
    {
      "group_key": "basic_info",
      "standard_field_key": "case_code",
      "tenant_field_key": "CASE_ID",
      "tenant_field_group": "basic_info",  // 甲方字段原始分组
      "mapping_status": "auto_mapped"
    }
  ],
  "extended_fields": [
    {
      "field_alias": "customer_level",
      "tenant_field_key": "CUSTOMER_LEVEL",
      "target_group_key": "basic_info",  // 拓展字段目标分组
      "source_group_key": "basic_info"   // 甲方字段原始分组
    }
  ],
  "total_count": 21,
  "mapped_count": 21
}
```

## 9. 与列表字段映射的区别

| 项目 | 列表字段映射 | 详情字段映射 |
|-----|------------|------------|
| 分组结构 | 无分组 | 有分组（核心） |
| 字段数量 | 15个 | 21个 |
| 视图模式 | 单一平铺视图 | 全部视图+分组视图 |
| 映射范围 | 全字段匹配 | 支持分组范围匹配 |
| 未使用字段 | 平铺显示 | 按分组显示 |
| 拓展字段 | 无分组 | 需指定目标分组 |
| 统计维度 | 字段总数 | 分组+字段 |
